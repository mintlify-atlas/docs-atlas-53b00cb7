This guide uses SQLite so you can skip any database or Redis setup and get straight to a running app.

<Note>
  `PAYMENT_WEBHOOK_SECRET` must not be empty. `PaymentWebhookController` reads it at boot, and several Artisan commands fail if it is missing. The minimal value `local-secret` used below is sufficient for local work.
</Note>

## Prerequisites

- PHP 8.2+
- Composer 2
- Node.js 20 and npm 10+

## Setup

<Steps>
  <Step title="Install PHP and JS dependencies">
    ```bash
    composer install
    npm install
    ```
  </Step>

  <Step title="Create your environment file">
    <CodeGroup>

    ```bash macOS / Linux
    cp .env.example .env
    ```

    ```powershell PowerShell
    Copy-Item .env.example .env
    ```

    </CodeGroup>

    Then open `.env` and set the following minimal variables:

    ```env .env
    APP_NAME="Vito Business OS"
    APP_URL=http://localhost:8000
    DB_CONNECTION=sqlite
    SESSION_DRIVER=database
    CACHE_STORE=database
    QUEUE_CONNECTION=database
    PAYMENT_PROVIDER=generic
    PAYMENT_WEBHOOK_SECRET=local-secret
    ```
  </Step>

  <Step title="Create the SQLite database file">
    <CodeGroup>

    ```bash macOS / Linux
    touch database/database.sqlite
    ```

    ```powershell PowerShell
    New-Item database/database.sqlite -ItemType File -Force
    ```

    </CodeGroup>
  </Step>

  <Step title="Generate the application key">
    ```bash
    php artisan key:generate
    ```
  </Step>

  <Step title="Run migrations and seed demo data">
    ```bash
    php artisan migrate --seed
    ```

    The seeder creates:

    - A super admin account (`admin@conectahn.com` / `password`)
    - A tenant owner account (`vendedor@test.com` / `password`)
    - 5 PRO tenants and 5 free tenants with products
    - 2 pending tenants awaiting approval
    - Categories, cities, departments, and municipalities
  </Step>

  <Step title="Link the storage directory">
    ```bash
    php artisan storage:link
    ```
  </Step>

  <Step title="Start all development processes">
    ```bash
    composer run dev
    ```

    This runs four processes concurrently via `concurrently`:

    | Process | Command |
    |---------|----------|
    | HTTP server | `php artisan serve` |
    | Queue listener | `php artisan queue:listen --tries=1` |
    | Log viewer | `php artisan pail --timeout=0` |
    | Vite dev server | `npm run dev` |
  </Step>
</Steps>

## What's running

After `composer run dev` starts, you have:

| URL | Surface |
|-----|---------|
| `http://localhost:8000` | Public site and all app routes |
| `http://localhost:8000/admin` | Filament super admin panel |
| `http://localhost:8000/app` | Filament tenant panel |
| `http://localhost:5173` | Vite HMR dev server (proxied automatically) |

The queue listener processes background jobs (reports, exports, emails, AI) in the same terminal session. The log viewer (`pail`) streams application logs.
